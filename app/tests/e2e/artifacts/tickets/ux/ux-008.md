# ux-008: Focus item selection non-deterministic across different equipment slots

## Summary

When a character has multiple Focus items across different equipment slots, `computeEquipmentBonuses()` applies the first one found via `Object.values(equipment)` iteration. The "winning" Focus depends on insertion order of the `EquipmentSlots` object, not an explicit slot priority.

Currently impossible in practice — all 5 catalog Focus items share the `accessory` slot, so equipping multiple Focus items requires custom API-crafted items in different slots.

## Affected Files

- `app/utils/equipmentBonuses.ts` (focusApplied guard, lines 38-53)

## Suggested Fix

If this becomes a real scenario (custom Focus items in non-accessory slots):
1. Define an explicit slot priority order (e.g., accessory > head > mainHand > offHand > feet > body)
2. Or add a UI warning when multiple Focus items are equipped: "Only one Focus active at a time (PTU p.295)"

## Impact

- **Current risk:** None — catalog prevents multiple Focus items
- **Future risk:** If custom item creation allows Focus in different slots, behavior is stable but implicit
