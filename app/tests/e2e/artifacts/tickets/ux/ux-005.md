# ux-005: Pokemon at full HP appears damaged after level-up

## Summary

When a Pokemon levels up via XP distribution, `maxHp` increases by `levelsGained` but `currentHp` is left unchanged. A Pokemon at full HP (e.g., `currentHp: 45, maxHp: 45`) that gains 3 levels ends up at `currentHp: 45, maxHp: 48` — appearing to be at 93.75% HP despite not having taken damage.

PTU does not explicitly prescribe whether `currentHp` increases on level-up. The current behavior is defensible (GM can heal manually), but creates a UX inconsistency.

## Affected Files

- `app/server/api/encounters/[id]/xp-distribute.post.ts`
- `app/server/api/pokemon/[id]/add-experience.post.ts`

## Suggested Fix

When `pokemon.currentHp === pokemon.maxHp` (full HP before level-up), also increase `currentHp` by `maxHpIncrease`. This preserves the "full HP" state without auto-healing damaged Pokemon.

```typescript
const wasAtFullHp = pokemon.currentHp >= pokemon.maxHp
const hpUpdate = {
  maxHp: pokemon.maxHp + maxHpIncrease,
  ...(wasAtFullHp ? { currentHp: pokemon.maxHp + maxHpIncrease } : {})
}
```

## Impact

Low — cosmetic UX issue. Pokemon that were at full HP before leveling will stay visually at full HP. Pokemon that were damaged stay at their current HP (correct behavior).
