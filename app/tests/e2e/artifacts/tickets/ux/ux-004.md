# ux-004: Wire getDisplayHp into VTTToken for player mode HP masking

## Summary

In player mode, VTTToken shows the exact HP bar fill for all combatants including enemies. The `getDisplayHp` and `getInfoLevel` utilities already exist in `usePlayerGridView.ts` but are not wired into VTTToken. A motivated player could deduce exact enemy HP from pixel-level analysis of the bar width.

The design spec's information asymmetry table specifies: Own = full info, Allied = name + exact HP, Enemy = name + HP percentage only.

## Affected Files

- `app/components/vtt/VTTToken.vue` — lines 157-163 (HP bar calculation)
- `app/composables/usePlayerGridView.ts` — `getDisplayHp` and `getInfoLevel` utilities (built but unused)

## Current Behavior

HP bar fill width is `entity.currentHp / entity.maxHp` for all combatants, showing exact proportional HP for enemies.

## Expected Behavior

In player mode, enemy token HP bars should use the percentage-based display from `getDisplayHp()` (which rounds to 25% increments: full, 75%, 50%, 25%, critical, fainted).

## Impact

Low visual impact since the HP bar is proportional (not numeric), and the same behavior exists on the Group View. PTU does not mandate HP hiding. However, the utility function exists and should be connected for design spec compliance.

## Suggested Fix

Pass `playerMode` and `getDisplayHp` to VTTToken. When in player mode and the combatant is an enemy, use the rounded percentage value instead of exact HP for the bar fill calculation. Defer to P2 implementation.
