# ux-007: Player's own tokens hidden in explored fog cells

## Summary

The `visibleTokens` filter in `usePlayerGridView.ts` hides ALL tokens in explored fog cells, including the player's own trainer and Pokemon tokens. If the GM sets a cell containing a player's own combatants to `explored` state, the player cannot see their own tokens on the grid.

The design spec (section 3.5) says explored cells show "Dimmed, terrain visible, no tokens" — but does not address whether own tokens should be exempt from this rule.

## Affected Files

- `app/composables/usePlayerGridView.ts` (visibleTokens filter at line ~90)

## Suggested Fix

Add an exception to the `visibleTokens` filter: if a combatant belongs to the current player, always show it regardless of fog state. This requires knowing the player's character ID in the composable.

Alternative: Always show own-side combatants (all Player-faction combatants) regardless of fog state.

## Impact

- **Information asymmetry:** Players lose track of their own tokens when fog state is `explored`
- **Current risk:** Low — GM typically reveals cells where player tokens stand. Only an issue if GM manually sets a player-occupied cell to `explored` instead of `revealed`
