# bug-031: Explored fog cells show tokens contrary to design spec

## Summary

The `usePlayerGridView.ts` composable shows tokens in both 'revealed' AND 'explored' fog cells. The design spec (section 3.5 information asymmetry table) explicitly states explored cells should show "Dimmed, terrain visible, no tokens." This leaks enemy position information that the GM may have intentionally obscured.

## Affected Files

- `app/composables/usePlayerGridView.ts` — lines 86-91 (`visibleTokens` filter)

## Current Behavior

```typescript
// Tokens visible in both 'revealed' and 'explored' cells
return state === 'revealed' || state === 'explored'
```

## Expected Behavior (per design spec)

```typescript
// Tokens only visible in 'revealed' cells
return state === 'revealed'
```

## Impact

Information leakage in fog-of-war scenarios. A player can see enemy token positions in explored-but-not-revealed cells, which the GM may have intentionally hidden. This is a design spec divergence, not a PTU rules issue (PTU has no fog-of-war mechanic).

## Suggested Fix

Either:
1. Change the filter to only show tokens in 'revealed' cells (`return state === 'revealed'`)
2. Or update the design spec to reflect the intended behavior that explored cells DO show tokens

Requires design author consultation to determine which option is correct.
