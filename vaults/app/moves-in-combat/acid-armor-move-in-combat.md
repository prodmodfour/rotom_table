Acid Armor appears in the [[encounter-act-modal-move-list]] as a purple (Poison-type) button showing "Acid Armor POISON Scene". No DB or AC values are displayed because it is a Status-class move.

The [[encounter-move-target-panel]] displays:
- Class: Status (no DB or AC fields shown)
- Range: Self, Set-Up
- Effect text describes the Liquefied state with its resolution effect (+1 Defense CS)

Because the range is "Self", only the user (Move Tester) is a valid target. The opposing combatant (Target Dummy) appears greyed out with "Out of range". There is no accuracy roll section since Status moves with the Self range auto-target.

The Set-Up keyword in the range indicates this is a two-part move (Set-Up then Resolution), but the app does not track the Liquefied intermediate state — the GM handles the multi-step resolution manually.
