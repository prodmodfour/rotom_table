# Moves In Combat (App Observations)

~220 observations of how individual moves and combat maneuvers appear in the encounter move target panel. Each note records what you see when a combatant uses that move/maneuver in the app.

## What you can't know without exploring here

- How each move's stat block actually renders in the target panel (damage base display, range text, effect text)
- Which moves have quirks or display issues in the current app
- How combat maneuvers (grapple, trip, push, etc.) present in the UI
- The actual data the app shows for a move vs what the PTR vault says it should show

## Naming convention

- `{move-name}-move-in-combat.md` — move observations (e.g. `absorb-move-in-combat.md`)
- `{maneuver-name}-maneuver-in-combat.md` — combat maneuver observations (e.g. `grapple-maneuver-in-combat.md`)

## Routing

- Looking up a **move's game stats**? Check `../../ptr/ptr_moves/`.
- Looking up **how a move should be implemented**? Check `../../documentation/moves/`.
- Looking up **how a move actually looks in the app right now**? You're in the right place.
