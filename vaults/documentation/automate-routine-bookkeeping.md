The app handles routine mechanical tracking so the GM can focus on narrative decisions. "Routine" means any operation where the rules fully determine a single correct outcome from known inputs — a damage formula, a stat recalculation, a movement path validation, a combat stage adjustment. If the system has all the information and there is no judgment call, it is routine.

This drives [[status-cs-auto-apply-with-tracking]] (Burn's -2 Def CS applied automatically), auto parse stat feature tags (detect and apply stat bonuses from feature text), and [[path-speed-averaging]] (calculate mixed-terrain movement automatically).

What is *not* routine: encounter design choices, item use approval in ambiguous situations, narrative pacing, anything where context or table mood matters. Those require the GM. Per [[gm-delegates-authority-into-system]], the dividing line is whether the rules alone can produce the correct outcome.
