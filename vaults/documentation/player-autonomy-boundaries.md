Player combat actions split into two categories based on whether the rules fully determine the outcome.

Direct actions — using a move, shifting, struggling, passing — are mechanically deterministic. The rules specify exactly what happens, so players execute them immediately without GM involvement. Requested actions — using an item, switching Pokemon, performing a maneuver — require GM judgment about context, timing, or validity, so they go through a request-approve flow.

This is distinct from [[server-enforcement-with-gm-override]]. Override is about the GM bypassing a rule the server enforces. Autonomy boundaries are about which actions need a human in the loop at all. The dividing line is judgment: if the action's legality and outcome can be fully resolved by rules, the player acts freely. If it requires situational assessment, the GM adjudicates.

Fluidity matters here. Direct actions like shifting happen potentially hundreds of times per combat — gating each behind GM approval would be tedious and would undermine the speed that [[automate-routine-bookkeeping]] is designed to protect. The GM sees all actions on [[the-table-as-shared-space]] and can undo or adjust via override if something exceptional requires it, but the default is uninterrupted play.
