PTU 1.05 does not include a formal system for habitat-based encounter tables with weighted species distributions. The encounter table feature in Rotom Table is an app-specific tool built to support GM workflow, not a PTU mechanic.

This distinction matters because the [[encounter-table-data-model]] and [[encounter-table-api]] must not claim PTU basis. The UI labels encounter tables as a GM tool, and the documentation describes them as app-specific rather than RAW-derived.

PTU does provide guidance on habitat types and which Pokemon appear where, but it does not formalize this into a weighted-random-encounter system. Rotom Table's implementation is an interpretation of that guidance, not a transcription of it.
