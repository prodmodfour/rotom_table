# Structural Patterns

[[design-patterns]] that assemble objects and classes into larger structures while keeping those structures flexible and efficient. They use composition and inheritance to form new functionality from existing pieces.

- [[adapter-pattern]] — makes incompatible interfaces work together
- [[bridge-pattern]] — separates abstraction from implementation so both can vary independently
- [[composite-pattern]] — composes objects into tree structures and treats them uniformly
- [[decorator-pattern]] — attaches new behaviors to objects dynamically via wrapping
- [[facade-pattern]] — provides a simplified interface to a complex subsystem
- [[flyweight-pattern]] — shares state across many similar objects to save memory
- [[private-class-data-pattern]] — encapsulates class attributes in a data object to control write access
- [[proxy-pattern]] — controls access to another object through a substitute
