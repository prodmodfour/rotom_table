When a game mechanic clears a condition (like switching clearing status effects) but the world state that caused the condition persists (like terrain), the app respects both: clear per the game mechanic, then re-apply on re-entry if the source is still active.

This is [[recall-clears-then-source-reapplies]]: PTR says switching clears Stuck/Slowed, so recall clears them. But if the Pokemon is sent back into the same rough terrain, the condition re-applies automatically. The clearing rule is respected AND the GM isn't burdened with manual re-application.
