# UML Structural View

A [[uml-view|view]] within [[uml-modeling-it-systems|IT system modeling]] that shows the static internal structure using [[uml-class-diagram|class diagrams]]. Documents all relevant classes, their relationships (associations), characteristics (attributes), and basic behavior (methods).

Class diagrams in the structural view represent [[uml-static-business-rule|static business rules]] — facts about the system that can be verified at any time.

## Construction approaches

- **Top-down analysis** — begins with general domain understanding to establish foundational class structure
- **Bottom-up analysis** — examines system inputs and outputs to identify required information

Both approaches combine during consolidation to produce the final class diagram.
