# Software Engineering Reference

~219 notes on general software engineering concepts used as a shared vocabulary throughout the documentation vault. These are not rotom_table-specific — they define patterns, smells, and techniques that the design notes reference.

## What you can't know without exploring here

- Precise definitions of SE concepts referenced by `[[wikilinks]]` in design notes (e.g. what exactly "shotgun surgery" or "combatant-as-lens" means when a design note links to `[[facade-pattern]]`)
- The full taxonomy of refactoring techniques, code smells, and design patterns available as design vocabulary

## Content categories

- `uml*.md` (~50) — UML 2.0 diagram types, modeling concepts, construction guides
- `*-pattern.md` (~30) — GoF and classic design patterns (Factory, Observer, Strategy, etc.)
- `*-smell.md` / `*-smells.md` (~29) — code smell catalog (Fowler/Kerievsky)
- `extract-*`, `replace-*`, `inline-*`, `introduce-*`, `remove-*`, etc. (~80) — refactoring techniques
- `solid-*.md`, `*-principle.md` (~7) — SOLID principles
- `refactoring*.md` (~10) — refactoring meta-notes (when to refactor, checklists, rules)
- `technical-debt*.md` (~10) — technical debt causes and concepts
- General: `clean-code.md`, `composition-over-inheritance.md`, `law-of-demeter.md`, `tell-dont-ask.md`, etc.
