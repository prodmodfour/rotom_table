# Design Patterns

Typical solutions to common problems in software design. A design pattern is not a specific piece of code but a general concept for solving a particular problem — a customizable blueprint adapted to the specifics of each program.

Patterns were popularized by the "Gang of Four" (Gamma, Helm, Johnson, Vlissides) in their 1994 book *Design Patterns: Elements of Reusable Object-Oriented Software*. They establish a shared vocabulary for developers to communicate about recurring structural challenges.

Design patterns differ from algorithms: an algorithm defines a clear set of actions to achieve a goal, while a pattern describes a higher-level approach whose exact implementation varies.
