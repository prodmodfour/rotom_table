# UML View

A view is a separate representation of a [[uml-model|model]] that emphasizes certain aspects while omitting others. Views exist because models containing all information simultaneously become unreadable.

Views interconnect — changes to one may require adjustments across all others. This is analogous to how adding a wing to a building affects floor plans, exterior views, and 3D models simultaneously.

Different views serve different audiences. Before creating views, consider:
- Business expertise level of the audience
- Required detail level
- Time availability for interpretation
- Language appropriateness (technical vs domain terminology)
- Abstraction level (clarity vs reusability)

Different model components with varying formality levels can serve different audiences while maintaining overall consistency.
