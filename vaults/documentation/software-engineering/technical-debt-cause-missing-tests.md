# Cause: Lack of Tests

Without immediate feedback from tests, quick but risky workarounds or kludges are encouraged. In worst cases, changes are deployed to production without any prior testing. The consequences can be catastrophic — an innocent-looking hotfix might send test emails to thousands of customers, or corrupt an entire database.

This is one of the most common sources of [[technical-debt]].
