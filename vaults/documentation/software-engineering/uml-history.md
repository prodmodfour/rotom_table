# UML History

Until the 1970s, software development was viewed as an artistic venture. As systems grew complex, this approach failed — the **software crisis**.

## Structured programming era

The crisis prompted structured programming methodologies:
- **Process-oriented methods** like HIPO (Hierarchy Input Processing Output) emphasized functional decomposition
- **Data-structure oriented methods** like the Jackson method derived program structure from data structures
- Early methods divided systems into data and procedures, using data flow charts, structure charts, and entity relationship diagrams

## Object-oriented evolution

Complex systems demanded maintainability and reusability. Object-oriented languages emerged in the 1970s–1980s, with analysis and design methodologies following in the 1990s. By mid-1990s, over 50 object-oriented methods existed.

## UML unification

In October 1994, Rational Software began unifying three major approaches:
- **Booch Method** (Grady Booch)
- **Object Modeling Technique / OMT** (James Rumbaugh)
- **OOSE** (Ivar Jacobsen)

What distinguishes UML is not its content but its **standardization to a single unified language with formally defined meaning**.

## Version timeline

- 1997: OMG approved version 1.1
- 1999: Version 1.3
- 2003: Version 1.5 and UML 2.0 Final Adopted Specification

Major companies including IBM, Oracle, Microsoft, and Hewlett-Packard participated in development.
