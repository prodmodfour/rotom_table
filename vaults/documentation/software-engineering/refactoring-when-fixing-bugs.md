# Refactoring When Fixing Bugs

Bugs behave like those in real life: they live in the darkest, dirtiest places in the code. Clean your code and the errors will practically discover themselves.

Managers appreciate proactive [[refactoring]] as it eliminates the need for special refactoring tasks later.
