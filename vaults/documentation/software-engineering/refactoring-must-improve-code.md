# Refactoring Must Improve Code

The code should become cleaner after [[refactoring]]. If it remains just as unclean, the effort was wasted.

This can happen when [[refactoring-in-small-changes|small changes are abandoned]] in favor of mixing many refactorings into one big change. It can also happen when working with extremely sloppy code — whatever you improve, the code as a whole remains a disaster.

When the code is too far gone, it may be worthwhile to completely rewrite parts of it. But before that, you should have written tests and set aside a good chunk of time. Otherwise the rewrite will itself become a source of problems.
