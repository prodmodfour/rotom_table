# Cause: Delayed Refactoring

A project's requirements constantly change, and at some point parts of the code become obsolete or cumbersome and must be redesigned to meet new requirements. Meanwhile, programmers write new code every day that works with the obsolete parts. The longer [[refactoring]] is delayed, the more dependent code will have to be reworked in the future.

A form of [[technical-debt]] where interest compounds as new code builds on outdated foundations.
