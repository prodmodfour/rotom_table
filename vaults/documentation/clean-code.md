# Clean Code

Code that has been structured to minimize long-term cost. [[refactoring]] transforms messy code into clean code. Clean code has four defining properties:

- **Obvious to other programmers.** No poor variable naming, bloated classes and methods, or magic numbers — nothing that makes code sloppy or difficult to grasp.
- **No duplication.** Every change to duplicated code must be made in every instance, increasing cognitive load and slowing progress.
- **Minimal moving parts.** Fewer classes and other parts mean less to keep in your head, less maintenance, and fewer bugs. Code is a liability — keep it short and simple.
- **Passes all tests.** Code with 95% of tests passing is dirty. Code with 0% test coverage is worse.

Clean code is easier and cheaper to maintain.
