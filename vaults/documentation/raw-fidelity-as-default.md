Rotom Table is its own game, built on a PTU 1.05 foundation. Every rule in Rotom Table is a Rotom Table rule — most happen to match PTU, but PTU is the starting point for design, not the authority.

When designing a new rule or mechanic, we research PTU 1.05 first. Where PTU is clear and the design is sound, Rotom Table adopts it. Where PTU is ambiguous, a [[per-conflict-decree-required]] makes the Rotom Table decision. Where PTU produces poor results, Rotom Table diverges with intention — see [[minimum-floors-prevent-absurd-results]].

The app uses [[server-enforcement-with-gm-override]] as its standard pattern — the system enforces Rotom Table's rules but provides an escape hatch for the GM.
