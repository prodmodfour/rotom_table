# Creational Patterns

[[design-patterns]] that control how objects are created. They abstract the instantiation process, making systems independent of how their objects are composed and represented. As systems evolve, creational patterns provide increasing flexibility about what gets created, who creates it, how, and when.

- [[factory-method-pattern]] — delegates object creation to subclasses via a creation method
- [[abstract-factory-pattern]] — produces families of related objects without specifying concrete classes
- [[builder-pattern]] — constructs complex objects step by step
- [[prototype-pattern]] — creates new objects by cloning existing ones
- [[singleton-pattern]] — ensures a class has only one instance with a global access point
- [[object-pool-pattern]] — manages reusable objects to avoid expensive repeated creation
