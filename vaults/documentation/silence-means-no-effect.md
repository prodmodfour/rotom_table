If PTR's rules do not mention an effect, it does not exist in Rotom Table. The app does not infer hidden mechanics from silence.

When PTR says nothing about a particular interaction — an ability affecting a status it doesn't name, a move having a secondary effect not listed in its text — the app treats that silence as intentional. No effect is implemented.

This prevents feature creep from assumed-but-unwritten rules. It also keeps the system auditable: every mechanic in the app traces to specific text, never to an inference about what PTR "probably meant."

If silence later turns out to be an oversight, [[cross-reference-before-concluding-omission]] ensures we check thoroughly before deciding, and [[per-conflict-decree-required]] creates a specific decree if action is needed.
