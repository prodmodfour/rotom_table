PTU's encounter creation workflow is narrative-first and GM-driven, not a random table roll. The steps: determine encounter purpose and narrative context, calculate level budget via [[encounter-budget-needs-ptu-basis]], select species from habitat, distribute levels across enemies, customize moves and [[trait-definition|traits]] based on significance.

The first step — figuring out *why* the players will be fighting — is critical. Aimless random encounters with no narrative purpose are explicitly discouraged. This is important context for [[ptu-has-no-formal-encounter-tables]] — the app's encounter tooling operationalizes qualitative guidance, it does not implement a RAW system.
