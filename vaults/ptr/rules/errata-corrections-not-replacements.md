The PTU errata is authoritative only for corrections and clarifications to existing rules. Content labeled as "playtest" or experimental does not automatically override the core rulebook.

The errata d20 capture system is labeled "Sept 2015 Playtest" and does not replace the [[core-capture-system-1d100]]. However, the 2016 playtest packet's version of No Guard was adopted via decree because it represents more recent design intent — but that required a [[per-conflict-decree-required]], not a blanket adoption.

Each playtest/errata conflict must be resolved individually. There is no general rule that "newer = better."
