Not every lore detail about a Pokemon needs a corresponding [[trait-definition|Trait]]. Traits should represent things with mechanical consequences in play. Lore notes in [[pokemon_ecology|pokemon ecology]] can describe behaviors, visual features, or ecological roles that have no mechanical expression.

[[Alolan Muk]]'s vivid coloration from chemical reactions is purely cosmetic — no trait needed. [[Barboach]]'s whisker-based hunting is described in lore but its Tremorsense trait already covers the mechanical reality.

A trait is warranted when it changes how the Pokemon interacts with the game. If the lore note alone is sufficient, the trait would be noise.

This coexists with the [[trait_philosophy|trait philosophy]] that aims to be exhaustive in describing what Pokemon *can do*. The distinction: cosmetic features and narrative flavor don't need traits; functional capabilities do.
