When the rules is ambiguous, silent, or produces poor results, Rotom Table makes its own design decision via a decree. There is no blanket rule that resolves all such cases the same way.

Each decree considers the rules source text, context, game balance implications, and precedent from related Rotom Table rulings. A decree for one [[trait-definition|trait]] (like [[no-guard-playtest-version]]) does not automatically apply to other traits — each must be evaluated on its own merits.

This ensures thoughtful, case-by-case design rather than sweeping generalizations that might break edge cases.
