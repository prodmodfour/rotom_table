# ux-003: QR code generation for SessionUrlDisplay

## Summary

The design spec P1 scope explicitly includes "Session URL display in GM View with QR code generation." The `SessionUrlDisplay` component shows URLs with copy-to-clipboard but has no QR code rendering. QR codes are specifically called out because the primary player device is a phone -- scanning a QR code from the GM's screen is substantially faster than typing a tunnel URL.

## Affected Files

- `app/components/gm/SessionUrlDisplay.vue`

## Current Behavior

SessionUrlDisplay shows LAN addresses and tunnel URL as text with copy-to-clipboard buttons. No QR code is rendered.

## Expected Behavior

Each URL (LAN and tunnel) should have an associated QR code that players can scan with their phone camera to quickly navigate to the player view.

## Suggested Fix

Use a lightweight QR code library (e.g., `qrcode` npm package for SVG/canvas generation, or `qrcode.vue` for a Vue component) to render QR codes inline next to or below each URL. The QR code should encode the full player-view URL (e.g., `http://192.168.1.50:3000/player` or `https://tunnel.example.com/player`).

- Used dark-theme-appropriate colors (foreground `#f0f0f5` matching `$color-text`)
