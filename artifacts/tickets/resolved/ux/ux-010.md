# ux-010: Overlapping significance tier boundary values

## Summary

The significance tier presets have overlapping boundaries at x5.0 and x7.0. Significant tier maxes at x5.0 and Climactic starts at x5.0; Climactic maxes at x7.0 and Legendary starts at x7.0. When the GM selects exactly x5.0 or x7.0, the value falls into both adjacent tiers.

## Affected Files

- `app/utils/encounterBudget.ts` — `SIGNIFICANCE_PRESETS` array

## Current Behavior

```
Significant:  { min: 4.0, max: 5.0 }
Climactic:    { min: 5.0, max: 7.0 }
Legendary:    { min: 7.0, max: 10.0 }
```

## Suggested Fix

Use exclusive upper bounds or half-step gaps:
- Significant: x4.0-x4.9, Climactic: x5.0-x6.9, Legendary: x7.0-x10.0
- Or use `<` for upper bound comparison in tier detection logic

## Impact

Low — the presets are used for defaults and the GM can override to any value. This is a UX polish concern that does not affect game logic correctness.

## Resolution Log

- **Commit:** 5d0ba76 — `fix: resolve overlapping significance tier boundaries at x5.0 and x7.0`
- **Files changed:** `app/utils/encounterBudget.ts` (2 lines changed)
- **Change:** Made upper bounds exclusive — Significant max 5.0 → 4.99, Climactic max 7.0 → 6.99. Each multiplier value now maps to exactly one tier.
