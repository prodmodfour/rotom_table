# bug-036: Player view 500 error — SCSS rgba(currentColor, 0.1) is not a valid color

## Summary

The Player view (`/player`) fails to render with a 500 Internal Server Error. The Vite SCSS compiler rejects `rgba(currentColor, 0.1)` because Sass's `rgba()` function requires an actual color value, not the CSS keyword `currentColor`.

## Error

```
[plugin:vite:css] [sass] $color: currentColor is not a color.
  ╷
112 │ background: rgba(currentColor, 0.1);
  │                       ^^^^^^^^^^^^^^^^^^^^^^^
  ╵
  pages/player/index.vue 112:19  root stylesheet
```

## Affected Files

- `app/pages/player/index.vue` line 471: `background: rgba(currentColor, 0.1);`

## Suggested Fix

Replace `rgba(currentColor, 0.1)` with a valid Sass color expression. Options:
1. Use a SCSS variable: `background: rgba($color-text, 0.1);`
2. Use CSS custom property with fallback: `background: color-mix(in srgb, currentColor 10%, transparent);`
3. Use a hardcoded color value that matches the intended design

## Impact

Player view is completely broken — renders a 500 error page. All player functionality is inaccessible.

## Pre-existing

This bug was NOT introduced by plan-20260227-122512. The file's last modification was prior to this collection. Discovered during Playwright smoke test.
