## Summary

`MoveTargetModal.vue` (line 288) passes `targetsRef` as both `targets` and `allCombatants` to `useMoveCalculation()`:

```ts
useMoveCalculation(moveRef, actorRef, targetsRef, targetsRef)
```

This means `allCombatants` only contains the selected targets, not all combatants in the encounter. Per decree-003, enemy-occupied squares count as rough terrain and should trigger a -2 accuracy penalty when targeting through them. But since `allCombatants` only has targets, non-target enemies along the line of sight are invisible to the rough terrain check.

## Affected Files

- `app/components/encounter/MoveTargetModal.vue` (line 288)

## Suggested Fix

Pass the full encounter combatant list from the encounter store instead of reusing `targetsRef`. The parent `GMActionModal.vue` has access to all combatants via the encounter store.

## Impact

Accuracy penalties for targeting through enemy-occupied squares (decree-003) are not applied when the enemy is not the target. This can result in incorrect accuracy calculations in multi-combatant encounters.

- **Commit:** b839636
