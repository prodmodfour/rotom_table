## Summary

The `app/server/api/game/new-day.post.ts:58-59` sets `boundAp: 0` during the new-day reset. This violates **decree-016** which states: "Extended rest clears only Drained AP, not Bound AP." Bound AP should persist until the binding effect ends (GM manual action), not be cleared automatically.

## PTU Rule

PTU p.252: "Extended rests... restore a Trainer's Drained AP." Decree-016 clarifies that Bound AP is separate and persists until the binding effect ends. Decree-019 says New Day is a "pure counter reset" — but boundAp is NOT a daily counter; it represents an active binding effect.

## Current Behavior

```js
// new-day.post.ts:58-59
drainedAp: 0,
boundAp: 0,  // WRONG — violates decree-016
currentAp: calculateMaxAp(level)  // WRONG — ignores existing boundAp
```

## Required Behavior

```js
// Keep boundAp unchanged, only clear drainedAp
drainedAp: 0,
// boundAp: unchanged (do NOT set to 0)
currentAp: calculateMaxAp(level) - existingBoundAp
```

**Note:** This requires changing from a batch `updateMany` to per-character updates, since each character has a different `boundAp` value.

## Affected Files

- `app/server/api/game/new-day.post.ts` — Remove `boundAp: 0`, change `currentAp` formula to subtract existing boundAp

## Decree Compliance

- **decree-016**: Currently VIOLATED. Fix restores compliance.
- **decree-019**: New Day remains a "pure counter reset" — drainedAp, restMinutesToday, injuriesHealedToday, and lastRestReset are all daily counters that should be reset. boundAp is NOT a daily counter.

## Ambiguity Note

The auditor flagged a possible ambiguity: "Is bound AP a daily counter that should be reset on New Day?" The strict reading of decree-016 says NO — bound AP persists until the binding effect ends. If the human disagrees, a decree can override this fix.
