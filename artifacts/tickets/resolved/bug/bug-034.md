# bug-034: @phosphor-icons/vue not installed — TerrainPainter and StatusConditionsModal fail

## Summary

Two components import from `@phosphor-icons/vue` but the package is not in `app/package.json` and not installed in `node_modules`. The dev server throws "Failed to resolve import" errors for both.

## Affected Files

- `app/components/vtt/TerrainPainter.vue` — imports `PhCircle`, `PhProhibit`, `PhWaves`, `PhShovel`, `PhWarning`, `PhArrowFatLineUp`, `PhCrosshairSimple`, `PhHourglass`
- `app/components/encounter/StatusConditionsModal.vue` — imports `PhWarningCircle`

## Suggested Fix

Run `npm install @phosphor-icons/vue` inside `app/`. This adds the Vue component library for Phosphor Icons, which is referenced in CLAUDE.md as the project's icon system.

## Impact

Build-blocking for VTT terrain editing and status condition management. These components cannot be loaded at all.

## Fix Log

- **479d0ee** — Added `@phosphor-icons/vue: ^2.2.1` to `app/package.json` dependencies (alphabetically ordered)
- Files changed: `app/package.json`
