# bug-033: SCSS mixin files fail — @use module scoping breaks variable resolution

## Summary

The build fails with "Undefined variable" errors across multiple components. The root cause: `nuxt.config.ts` uses `@use` in `additionalData` to inject SCSS partials (`_modal.scss`, `_pokemon-sheet.scss`, `_difficulty.scss`, `_sheet.scss`), but `@use` creates module-scoped boundaries. Variables from `_variables.scss` are available in the component scope but NOT inside the mixin files themselves, because each `@use`d file is its own isolated module.

With `@import` (deprecated), files were concatenated and shared scope. With `@use`, each module must explicitly declare its own dependencies.

## Affected Files

Mixin files that reference `_variables.scss` variables without `@use`-ing them:
- `app/assets/scss/_modal.scss` — uses `$z-index-modal`, `$color-bg-secondary`, `$spacing-*`, `$glass-*`, `$border-radius-*`, `$color-text*`, `$shadow-xl`, `$color-accent-violet`
- `app/assets/scss/_pokemon-sheet.scss` — uses `$type-*`, `$spacing-*`, `$font-size-*`, `$color-*`, `$border-radius-*`, `$glass-border`
- `app/assets/scss/_difficulty.scss` — uses `$color-neutral`, `$color-success`, `$color-info`, `$color-warning`, `$color-danger`
- `app/assets/scss/_sheet.scss` — uses `$spacing-*`, `$glass-*`, `$color-*`, `$border-radius-*`, `$font-size-*`, `$transition-*`, `$gradient-*`

Components that fail as a result (non-exhaustive — any component using these mixins):
- `components/gm/EncounterHeader.vue` (btn-with-icon)
- `components/encounter/GMActionModal.vue` (modal-overlay-enhanced)
- `components/encounter/AddCombatantModal.vue` (modal-overlay-enhanced)
- `components/encounter/XpDistributionModal.vue` (modal-overlay-enhanced)
- `components/encounter-library/LoadTemplateModal.vue` (modal-container-base)
- `components/encounter-library/SaveTemplateModal.vue` (modal-container-base)
- `components/vtt/VTTContainer.vue` (btn-with-icon)
- `components/gm/NewEncounterForm.vue` (btn-with-icon)
- `components/group/PlayerLobbyView.vue` (type-color-modifiers)
- `components/group/CombatantDetailsPanel.vue` (pokemon-sheet-type-badge)

## Suggested Fix

Add `@use "variables" as *;` at the top of each mixin file. Sass deduplicates `@use` calls — the same module loaded from both `additionalData` and internally won't conflict.

Also check `app/assets/scss/components/` partials for the same issue (they're `@import`ed by components, not loaded via `additionalData`).

## Impact

Build-blocking. The dev server starts but every page that renders a component using these mixins throws a pre-transform error. No pages with modals, encounter UI, VTT, or Pokemon sheets can render.

The `components/_index.scss` uses `@forward` but is not imported anywhere, so its forwarded partials are not at risk. No additional fixes needed.
