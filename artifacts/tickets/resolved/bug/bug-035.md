# bug-035: LevelUpNotification.vue references 6 missing phosphor SVG icons

## Summary

`LevelUpNotification.vue` uses `<img src="/icons/phosphor/...">` tags for 6 SVG icons that don't exist in `app/public/icons/phosphor/`. The dev server throws "Failed to resolve import" errors.

## Missing Icons

- `/icons/phosphor/star.svg`
- `/icons/phosphor/arrow-right.svg`
- `/icons/phosphor/graduation-cap.svg`
- `/icons/phosphor/sword.svg`
- `/icons/phosphor/lightning.svg`
- `/icons/phosphor/arrow-circle-up.svg`

45 other SVGs exist in the directory — these 6 were never added.

## Affected Files

- `app/components/encounter/LevelUpNotification.vue`

## Suggested Fix

Option A: Convert to `@phosphor-icons/vue` components (once bug-034 is resolved), matching the pattern used by `TerrainPainter.vue` and `StatusConditionsModal.vue`.

Option B: Download the 6 missing SVGs from Phosphor Icons and place them in `app/public/icons/phosphor/`.

Option A is preferred for consistency — the project should converge on one icon approach.

## Impact

Build-blocking for the level-up notification after XP distribution. The component cannot be loaded.

Updated SCSS to remove img-specific filter hacks in favor of `currentColor` inheritance.
