## Summary

When a character gains enough XP to auto-level past a milestone level, the LevelUpModal is not prompted for milestone choices. The system silently skips the milestone, applying level-up stat changes but losing the human decision points (e.g., choosing a feature, edge, or stat specialization at milestone levels).

## Problem

PTU milestone levels (5, 10, 20, 30, 40) require player/GM choices. The auto-level pathway does not pause to collect these choices. Three sub-problems:

1. **Race condition**: Server updates character level before LevelUpModal opens. If character data is reloaded, `character.level` equals the new level, causing the modal to compute zero advancement.
2. **No auto-save**: Level-up choices from the modal were stored in `editData` but not persisted to the server unless the user was in edit mode.
3. **Missing modal trigger**: Encounter XP distribution and Quest XP dialog award XP without ever triggering the LevelUpModal.

## Suggested Fix

When auto-leveling through a milestone level, queue a pending milestone decision that must be resolved before the character can proceed. The LevelUpModal should be shown for each milestone level passed through.

## Impact

Players may miss milestone features/edges, resulting in underpowered characters compared to PTU RAW.
