## Summary

The mount and dismount endpoints broadcast `encounter_update` WebSocket events with `{ encounterId: id }` instead of the full encounter response. Client-side `updateFromWebSocket(data: Encounter)` in `encounter.ts` iterates `data.combatants`, which is `undefined` when the payload is `{ encounterId: id }`, causing a `TypeError` that crashes WebSocket state sync for all group and player clients.

## Problem

Same malformed broadcast pattern found in Living Weapon endpoints (code-review-297 C1). The Living Weapon fix cycle will fix the new instances, but mount.post.ts and dismount.post.ts have the same pre-existing bug.

## Suggested Fix

Send the full encounter response as the broadcast data:
```typescript
broadcastToEncounter(id, {
  type: 'encounter_update',
  data: response  // full buildEncounterResponse() result
})
```

## Impact

Every mount/dismount action crashes WebSocket state sync for group and player views. Multi-view setups see stale mount/dismount state.
