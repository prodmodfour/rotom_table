## Summary

The capture attempt endpoint (`app/server/api/capture/attempt.post.ts`) does not check `pokemon.ownerId` before proceeding with the capture roll. An already-owned Pokemon can be captured, overwriting the existing owner. PTU capture rules target "wild Pokemon" specifically.

## PTU Rule

PTU capture rules (p.214) describe capturing wild Pokemon. Poke Balls should not work on already-owned Pokemon. Attempting to capture a trainer's Pokemon is not a valid action.

## Current Behavior

```
attempt.post.ts:24-33 — Fetches the Pokemon record but does NOT check pokemon.ownerId.
On success (line 100), the existing ownerId is overwritten to the new trainer.
```

## Required Behavior

After fetching the Pokemon (line 33), add ownership validation:
```js
if (pokemon.ownerId) {
  throw createError({ statusCode: 400, message: 'Cannot capture an owned Pokemon' })
}
```

## Affected Files

- `app/server/api/capture/attempt.post.ts` — Add ownership check after Pokemon fetch

## Impact

HIGH — Allows "stealing" Pokemon from other trainers via capture, which PTU does not permit. Simple 3-line fix.
