# bug-049: Remove dead 'Bound' condition checks from switching system

## Summary

The switching system checks for a 'Bound' condition that doesn't exist in the StatusCondition type union, is never inflicted by any code, and has no PTU basis. Per decree-044, only Trapped blocks recall.

## Problem

Four locations check for `'Bound'` alongside `'Trapped'` when validating recall/switch:
1. `switching.service.ts` line ~429: `validateSwitch` — `allRecalledConditions.includes('Bound')`
2. `switching.service.ts` line ~612: `validateForcedSwitch` — `allRecalledConditions.includes('Bound')`
3. `useSwitching.ts` line ~155: `canForcedSwitch` — `allConditions.includes('Bound')`
4. `recall.post.ts` line ~119: recall endpoint — `allConditions.includes('Bound')`

Additionally, `switching.service.test.ts` has a test case using `['Bound'] as any` to bypass TypeScript.

## Required Implementation

1. Remove `|| allRecalledConditions.includes('Bound')` from all four check locations
2. Remove or rewrite the 'Bound' test case in `switching.service.test.ts` (delete the `as any` cast test)
3. Verify no other files reference 'Bound' as a status condition

## Notes

- Per decree-044, Grapple (PTU RAW) does not block recall — only Trapped does
- Vortex moves (Fire Spin, Whirlpool, etc.) already inflict Trapped, which is correctly handled

## Resolution Log

- **0caab34e** `fix: remove dead 'Bound' condition checks from switching system (decree-044)` — Removed `|| allConditions.includes('Bound')` from all 4 code locations and deleted the `['Bound'] as any` test case
- **02beecb7** `fix: read tempConditions from combatant, not entity, in recall.post.ts` — Also removed Bound check from recall.post.ts as part of the tempConditions fix
