## Summary

When exporting and re-importing encounter tables, the density field is lost in the round-trip. Additionally, species not present in the local database are silently dropped on import rather than flagged.

## Suggested Fix

Include density in the export format. On import, warn about unrecognized species rather than silently dropping them.

## Impact

Data loss on encounter table round-trip. Silent species removal could confuse GMs.

## Resolution Log

### Fix 1: Include density in export (45494227)
- `app/server/api/encounter-tables/[id]/export.get.ts` -- added `density: table.density` to export format

### Fix 2: Restore density on import (02d39a26)
- `app/server/api/encounter-tables/import.post.ts` -- added `density` to ImportData interface, validated against valid tiers (falls back to 'moderate'), passed to Prisma create, included in response

### Fix 3: Surface species warnings in UI (ff2d6f1c)
- `app/components/encounter-table/ImportTableModal.vue` -- when unrecognized species produce warnings, the modal now stays open with a "Continue" button so the user can read the warning before navigating away

### Fix 4: Test coverage (4fd86abe)

### Fix 5: Density fallback tests (b16fd6bc)
