## Summary

Both encounter table editor pages (`/gm/habitats/:id` and `/gm/encounter-tables/:id`) render with empty `<main>` content because the template uses `<EncounterTableTableEditor>` but Nuxt's auto-import with `pathPrefix: false` resolves the component as `<TableEditor>`.

## Problem

The component is registered as `TableEditor` (from `components/encounter-table/TableEditor.vue`) but the page templates reference it as `EncounterTableTableEditor` (the path-prefixed name). With `pathPrefix: false` in nuxt.config.ts, the path-prefixed name is not registered.

## Suggested Fix

Rename `<EncounterTableTableEditor>` to `<TableEditor>` in both page files.

## Impact

- 8 encounter-tables capabilities blocked (C004, C016, C017, C020, C021, C022, C026, C027)
- The encounter table editor (core editing UI for the domain) is completely inaccessible
