# bug-050: Server-side calculate-damage missing moveKeywords passthrough

## Summary

The `calculate-damage.post.ts` endpoint does not pass `moveKeywords: move.keywords` to the `calculateDamage()` function, despite the `DamageCalcInput.moveKeywords` field being available. This means the server-side 9-step damage calculation will not skip STAB for Living Weapon moves with the `[Weapon]` keyword.

## Problem

PTU p.287 states weapon moves "can never benefit from STAB." The client-side path correctly blocks STAB via the `moveKeywords` field (fixed in commit b3dcb764), but the server-side endpoint at `calculate-damage.post.ts` lines 273-289 does not pass `moveKeywords` through to `calculateDamage()`.

## Impact

**Currently not triggerable** — the existing Living Weapon species (Honedge/Doublade/Aegislash: Steel/Ghost) grant Normal-type weapon moves, so STAB type matching never occurs. However, any future Living Weapon species with type-matching weapon moves would receive incorrect STAB damage from the server-side calculation.

## Fix

Single-line addition: add `moveKeywords: move.keywords,` to the `calculateDamage` input object in `calculate-damage.post.ts`.

## Notes

The `move` object is resolved from `getEffectiveMoveList()` (line 192), which already sets `keywords: ['Weapon']` for Living Weapon moves.
