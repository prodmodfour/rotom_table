## Summary

The PlayerPokemonCard summary renders correctly (sprite, name, types, level, HP bar), but expanding to show details triggers `TypeError: Cannot read properties of undefined (reading 'length')` at PlayerPokemonCard.vue line 318. This blocks viewing stats, abilities, moves, and capabilities for team Pokemon.

## Problem

The expansion code accesses `pokemon.statusConditions` or `pokemon.abilities` without null guards. The player-view API response may not always include these fields (they could be undefined when no conditions/abilities exist).

## Suggested Fix

1. Ensure the player-view API response (C011) always returns `statusConditions` and `abilities` as arrays (even if empty)
2. Add `?? []` fallback in the component template for defensive rendering

## Impact

- C025 PlayerPokemonCard expansion broken (MEDIUM)
- C026 PlayerMoveList blocked as a consequence (rendered inside expanded card)
- Players cannot view their Pokemon's detailed stats, abilities, or moves on the Team tab
