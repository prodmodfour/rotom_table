# bug-032: No levelMin <= levelMax validation in encounter tables

## Summary

No cross-validation exists in table or entry creation APIs for level range fields. If `levelMin > levelMax`, the level generation formula produces a negative range, resulting in unpredictable levels.

## Affected Files

- `app/server/api/encounter-tables/` (creation and update endpoints)

## Suggested Fix

Add Zod validation: `levelMin` must be <= `levelMax`. Return 400 error if violated.

## Impact

Edge case — only occurs if GM manually enters invalid level ranges. Results in unpredictable Pokemon levels.
