## Summary

Two related issues prevent vision state from syncing across clients:

1. **CRIT-1:** `updateFromWebSocket` in encounter store does not propagate `visionState` on combatants. When one GM client toggles Darkvision, other connected clients silently drop the vision state during the surgical WebSocket update, causing stale vision indicators and incorrect accuracy penalty calculations.

2. **HIGH-1:** The vision toggle API endpoint does not call `broadcastToEncounter()` after persisting the change. Every other encounter mutation endpoint broadcasts; this one does not. Even with CRIT-1 fixed, no WebSocket event is emitted to deliver the data.

## Required Fix

1. Add `broadcastToEncounter()` call in `vision.post.ts` after successful persistence
2. Update `updateFromWebSocket` handler in encounter store to merge `visionState` field on combatants
3. Both fixes are required together — either alone is insufficient

## Impact

High — multi-client encounters will show inconsistent vision state and potentially calculate different accuracy penalties on different clients.
