## Summary

Two decree violations in AP management endpoints:

1. **decree-016 violation**: Extended rest endpoint clears ALL `boundAp` instead of clearing only drained AP. decree-016 states: "Extended rest clears ONLY drained AP." Also missing duration parameter per decree-018.

2. **decree-028 violation**: New Day endpoints (`[id]/new-day` and `global/new-day`) reset `boundAp` to 0. decree-028 states: "Bound AP persists across New Day."

## Problem

Both violations cause incorrect AP state for players, silently removing bound AP that should persist.

## Suggested Fix

1. Extended rest: Clear only the "drained" portion of boundAp, not all bound AP. Implement decree-018 duration parameter.
2. New Day: Stop resetting `boundAp` to 0. Bound AP should persist across new day boundaries.

## Impact

- Incorrect AP economy for all characters using AP-binding abilities
- Silent data loss on extended rest and new day transitions
