# bug-048: 'Bound' condition check blocks recall without PTU basis

## Summary

All Trapped checks in the switching system also check for `allConditions.includes('Bound')`. However, 'Bound' is not in the `StatusCondition` type union, and PTU's "Bound" (from Destiny Bond, p.367) is a revenge-faint mechanic — NOT a movement/recall restriction.

Both code-review-308 and rules-review-281 independently flagged this:
- **Code perspective:** 'Bound' is dead code that bypasses the type checker only via `as any` in tests
- **Rules perspective:** If a Pokemon is Bound by Destiny Bond but NOT Trapped, the code would incorrectly prevent its recall — a false positive blocking legal switches

## Related

See decree-need-043 for human ruling on whether 'Bound' was intended as a house rule synonym for a Trapped-like condition.

## Suggested Fix

Pending decree-need-043 ruling:
- If 'Bound' has no intended meaning: remove all 'Bound' checks from switching system
- If 'Bound' is intended as a house rule: add to StatusCondition type union and document
