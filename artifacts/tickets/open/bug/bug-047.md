# bug-047: Friend Ball loyalty fallback default mismatch

## Summary

In `app/server/api/capture/attempt.post.ts` line 192, the nullish coalescing fallback for loyalty is `2` (Wary), but `schema.prisma` declares `loyalty Int @default(3)` (Neutral). The fallback should be `3` to match the schema default.

## Problem

```typescript
loyalty: (pokemon.loyalty ?? 2) + 1  // fallback is 2 (Wary)
```

The schema default is 3 (Neutral). If `pokemon.loyalty` is somehow null/undefined, the fallback produces a different starting value than the database would assign.

## Impact

Unlikely to trigger in practice since Pokemon records always have loyalty set by the generator. But the mismatch is confusing and could cause issues if the code path is ever exercised.

## Suggested Fix

Change fallback from `2` to `3` to match schema default.
