## Summary

The character POST and PUT API endpoints do not enforce a maximum trainer level of 50. A GM can set level > 50 via direct API calls or edge-case UI interactions.

## Suggested Fix

Add server-side validation on character create/update to reject level values > 50.

## Impact

LOW in practice (GM-only action, unlikely accidental), but violates PTU core rules.

## Resolution Log

### Duplicate Code Path Check

All code paths that set or modify trainer level were audited:

1. **POST /api/characters** — Was unvalidated. Now uses `validateTrainerLevel()`.
2. **PUT /api/characters/:id** — Was unvalidated. Now uses `validateTrainerLevel()`.
3. **POST /api/characters/:id/xp** — Already safe via `applyTrainerXp()` which caps at `TRAINER_MAX_LEVEL`.
4. **POST /api/encounters/:id/trainer-xp-distribute** — Already safe via `applyTrainerXp()`.
5. **POST /api/capture/attempt** (new species XP) — Already safe via `applyTrainerXp()`.
6. **POST /api/pokemon/:id/evolve** (new species XP) — Already safe via `applyTrainerXp()`.
7. **CSV import** (`csv-import.service.ts`) — Was unvalidated. Now uses `validateTrainerLevel()`.
