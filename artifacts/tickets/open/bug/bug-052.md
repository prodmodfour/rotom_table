## Summary

`PlayerCharacterSheet.vue` renders feature tags with bare `class="tag"` instead of the correct `tag--feature` variant class. This causes features to display with neutral/default styling instead of the teal color defined in `_tags.scss`.

## Problem

All other views (GM character detail, HumanClassesTab) correctly apply `tag--feature` to feature tags. Only the player character sheet is missing the variant class. This is a pre-existing issue that predates the refactoring-096 CSS harmonization work — it was surfaced during the re-review (rules-review-312).

## Suggested Fix

Add the `tag--feature` class to feature tag elements in the player character sheet template.

## Impact

Visual inconsistency only. Features on the player sheet appear with neutral tag styling instead of the teal color used everywhere else.
