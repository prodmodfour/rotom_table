## Summary

The character detail page (`/gm/characters/[id]`) fails to render due to an SCSS compilation error: `[sass] Undefined variable: $spacing-xs` at `_level-up-shared.scss:82`. This blocks all 15 character detail tab components (C056-C065, C077-C083) and the page itself (C091).

The same root cause also blocks combat equipment components (combat-C136 HumanEquipmentTab, combat-C137 EquipmentCatalogBrowser) since they live on the character sheet page.

## Problem

`_level-up-shared.scss` references `$spacing-xs` without the global SCSS variables being imported into scope. Vite's scoped SCSS compilation does not auto-inject the variables partial for `@use`d files.

## Suggested Fix

Either:
1. Add `@use '../_variables' as *;` at the top of `_level-up-shared.scss`
2. Or ensure Vite `css.preprocessorOptions.scss.additionalData` includes the variables for all SCSS blocks

## Impact

- 15 character-lifecycle capabilities blocked (Error classification in browser audit)
- 2 combat capabilities blocked (Absent classification in browser audit)
- Character detail page completely inaccessible to GM
