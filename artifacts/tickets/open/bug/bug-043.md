## Summary

The `rollAccuracyCheck()` function in `useCapture.ts` rolls 1d20 but never compares the result against AC 6 to determine hit/miss. The caller always proceeds to `attemptCapture()` regardless of the roll result. Every Poke Ball throw automatically hits.

## PTU Rule

PTU p.214: "Poke Balls can be thrown as a Standard Action, as an AC6 Status Attack Roll." If the d20 roll is below 6, the ball misses the target and no capture roll should occur.

## Impact

Overpowered capture mechanics — balls never miss. Per PTU, approximately 25% of basic throws (rolling 1-5 on d20) should miss. This has been the behavior since capture was first implemented, so it affects all existing capture flows (GM-initiated and player-initiated).

## Affected Files

- `app/composables/useCapture.ts` — `rollAccuracyCheck()` (line ~227-234) returns raw d20 but no AC comparison
- `app/composables/usePlayerRequestHandlers.ts` — `handleApproveCapture` calls `rollAccuracyCheck()` then always proceeds

## Suggested Fix

After `rollAccuracyCheck()`, check if the roll meets AC 6. If not, return a "miss" result instead of proceeding to `attemptCapture()`. Handle natural 1 (always miss) and natural 20 (always hit + crit bonus) per PTU rules.

```typescript
const accuracyResult = rollAccuracyCheck()
if (accuracyResult.roll < 6 && accuracyResult.roll !== 20) {
  // Ball missed — no capture attempt
  return { hit: false, roll: accuracyResult.roll }
}
```
