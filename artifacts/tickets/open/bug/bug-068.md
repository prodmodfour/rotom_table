## Summary

The `densityMultiplier` field on the `TableModification` model is not included in encounter table export/import, causing data loss on round-trip.

## Problem

Same class of bug as bug-060 (density field lost on export/import), but for the `densityMultiplier` field on `TableModification` records rather than the base table density. When exporting a table with modifications that have custom density multipliers, the multiplier values are silently dropped.

## Suggested Fix

Add `densityMultiplier` to the modification serialization in the export endpoint, and restore it during import.

## Impact

- Data loss on round-trip export/import of encounter tables with modified density multipliers
- Affects GMs who export/import modified encounter tables
