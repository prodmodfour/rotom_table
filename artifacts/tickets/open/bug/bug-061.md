## Summary

The AP drain injury healing pathway does not validate that the character has >= 2 AP before allowing the drain. At 0-1 AP, the drain can over-reduce AP below zero.

## Suggested Fix

Add validation: if character AP < 2, reject the AP drain healing action with an appropriate error.

## Impact

Edge case — only occurs when healing injuries at very low AP.

## Resolution Log

- `2daea9f1` — fix: validate AP >= 2 before allowing AP drain injury healing (`app/server/api/characters/[id]/heal-injury.post.ts`)
- `4e4f77f3` — fix: disable drain AP button when currentAp < 2 and show AP in status (`app/components/common/HealingTab.vue`)

### Changes Made

**Server (heal-injury.post.ts):** Added early validation in the `drain_ap` path — if `character.currentAp < 2`, returns `{ success: false }` with descriptive error message before any DB write.

**Client (HealingTab.vue):** Added `currentAp` computed property. Drain AP button disabled when `currentAp < 2`. Added Current AP to status display with red highlight when below drain threshold.

### Duplicate Code Path Check

Grepped for all AP drain operations (`drainedAp + 2`, `currentAp - 2`). Only one code path performs AP drain: `characters/[id]/heal-injury.post.ts`. Other references to `drainedAp` are resets (new-day, extended-rest) or reads (serializers, scene AP restore). No duplicate paths found.
