# ux-016: hasActed flag not set when all three actions individually exhausted

## Summary

`pass.post.ts` sets `hasActed: true` alongside all three action flags (standardUsed, shiftUsed, swiftUsed), signaling turn completion. The new `action.post.ts` (from bug-044) only sets individual action flags.

If a combatant uses all three actions via separate calls (standard via action.post, shift via another endpoint, swift via another), `hasActed` stays `false`. The GM UI may show the combatant as still having an active turn even though all actions are exhausted.

## Impact

Edge case — most turns end via pass.post.ts which sets hasActed. Only affects cases where all three action types are consumed individually without a pass action.

## Suggested Fix

Either:
1. Check if all three actions are consumed after setting any individual flag, and auto-set `hasActed: true` if so
2. Change turn completion detection to check individual flags rather than relying solely on `hasActed`
